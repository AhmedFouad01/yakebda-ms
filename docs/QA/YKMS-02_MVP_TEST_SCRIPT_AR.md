# سيناريو اختبار يدوي — YKMS-02 MVP — يا كبدة (YAKEBDA MS)

يُنفذ بالترتيب. أي خطوة لا تحقق «النتيجة المتوقعة» تعتبر فشلًا يجب تسجيله.

| # | الخطوة | الأمر / الإجراء | النتيجة المتوقعة |
|---|---|---|---|
| 1 | تثبيت الحزم | `npm install` في جذر المشروع | ينتهي بدون أخطاء |
| 2 | إعداد البيئة | `./scripts/setup-db.sh` ثم `cp apps/api/.env.example apps/api/.env` | قاعدتا `ykms` و `ykms_test` جاهزتان |
| 3 | تشغيل الترحيلات | `npm run api:migrate` | ظهور `20260705_001_foundation` و `20260709_002_ykms_02_restaurant_mvp` |
| 4 | زرع البيانات | `npm run api:seed` | طباعة بيانات الدخول التجريبية |
| 5 | تشغيل الـ API | `npm run api:dev` | `/api/v1/health` يرجع `dir: rtl` واسم YAKEBDA MS |
| 6 | تشغيل الواجهة | `npm run admin:dev` ثم فتح `http://localhost:5173` | صفحة دخول عربية RTL بشعار «يا كبدة» |
| 7 | دخول الإدارة | `owner@ykms.local / Owner@12345` | الدخول للوحة التحكم |
| 8 | لوحة التحكم | فتح «لوحة التحكم» | بطاقات مبيعات اليوم وعدد الطلبات بأرقام حقيقية من الـ seed |
| 9 | إنشاء فئة | المنيو ← الفئات ← إضافة «حلويات» | تظهر في الجدول فورًا |
| 10 | إنشاء صنف | المنيو ← الأصناف ← إضافة «أرز بلبن» في «حلويات» بسعر 20 | يظهر بالجدول ويمكن تعديل سعره |
| 11 | إتاحة الفرع | المنيو ← أسعار وإتاحة الفروع ← «فرع تجريبي» | «طبق كبدة» يظهر غير متاح وسعر «ساندوتش كبدة» = 28 |
| 12 | فتح نقطة البيع | القائمة ← «نقطة البيع» | شبكة أصناف بفئات عربية وسلة فارغة |
| 13 | طلب تيك أواي | إضافة «ساندوتش كبدة» (كبير + جبنة) ×2 و«بيبسي»، خصم 5، دفع نقدي، إرسال | رسالة «تم إنشاء الطلب رقم …» وإجمالي 85 ومعاينة إيصال |
| 14 | إرسال للمطبخ | فتح «المطبخ» | الطلب في عمود «تم الإرسال»؛ زر «بدء التحضير» ينقله لـ«في المطبخ» |
| 15 | جاهز | زر «جاهز» على البطاقة | الطلب ينتقل لعمود «جاهز» |
| 16 | إتمام الدفع | «الطلبات» ← تفاصيل طلب غير مدفوع ← «تسجيل دفعة» | الدفعة تظهر في الإيصال والحالة تتحول لـ«مكتمل» بعد الضغط |
| 17 | طباعة الإيصال | زر «طباعة الإيصال» في تفاصيل الطلب | رسالة نجاح، ومهمة جديدة `pending` في «طابور الطباعة» بسطر «شكرًا لاختيارك يا كبدة» |
| 18 | التقارير | فتح «التقارير» | مبيعات اليوم، أفضل الأصناف، طرق الدفع، المبيعات حسب الفرع — كلها بأرقام |
| 19 | دخول المطبخ | خروج ثم دخول `kitchen@ykms.local / Kitchen@12345` | الدخول يعمل وشاشة «المطبخ» تعرض الطلبات |
| 20 | تقييد الصلاحيات | بنفس المستخدم: محاولة إضافة فئة أو فتح التقارير | رسالة «ليست لديك صلاحية…» (403) |
| 21 | طلب طاولة | دخول admin ← «الطاولات» ← «فتح طلب» على طاولة متاحة ← إرسال طلب صالة | نقطة البيع تفتح بالطاولة محددة، وبعد الإرسال تتحول الطاولة لـ«مشغولة» |
| 22 | طلب دليفري بعميل | نقطة البيع ← «دليفري» ← اختيار «أحمد محمد» ← رسوم توصيل 10 ← إرسال | العنوان يُملأ تلقائيًا والإيصال يعرض العميل ورسوم التوصيل |
