## Milestone Log — YKMS-01 + YKMS-01H

**التاريخ:** 2026-07-05
**الحالة:** ✅ Completed — مكتملة بعد إنهاء تنظيف التسمية والتقوية الأمنية

### ما تم تسليمه

| البند | التفاصيل |
|---|---|
| هيكل المشروع | Monorepo بمساحات عمل: `apps/api`، `apps/admin`، `packages/bridge-contract`، `docs/`، `scripts/` |
| أساس الـ API | Express + TypeScript + Knex + PostgreSQL — كل المسارات تحت `/api/v1`، رسائل خطأ عربية |
| المصادقة ودخول PIN | دخول الإدارة بالبريد وكلمة المرور (FR-011) + دخول الكاشير/الويتر بـ PIN على مستوى الفرع (FR-012) — JWT |
| RBAC | 8 أدوار نظامية + أدوار مخصصة وصلاحيات لكل دور، مع `requirePermission` على المسارات الحساسة (FR-010/FR-013) |
| الفروع | CRUD كامل بمنطقة زمنية وحالة (FR-001) |
| الأجهزة | تسجيل الأجهزة بأنواعها الستة + Device Profiles بإعدادات JSON (FR-002/FR-004) |
| نقاط الهاردوير | طابعات إيصالات/مطبخ، درج كاش، شاشة عميل، باركود — USB/LAN/Bluetooth/Windows Driver، ESC/POS (FR-070) |
| مهام الطباعة | دورة حياة `pending → printing → printed | failed` مع عداد محاولات، والفشل لا يوقف التشغيل (FR-071، NFR-002) |
| عقد الجسر | `packages/bridge-contract`: heartbeat كل 15 ثانية، سحب مهام كل ثانيتين، إبلاغ نتيجة — CONTRACT.md عربي + أنواع TypeScript |
| عملاء ورموز API | إنشاء عملاء (bridge/website/qr/mobile)، رموز بنطاقات scopes تُعرض مرة واحدة وتُخزن sha256 فقط (FR-161/162، NFR-003) |
| سجل العمليات | Audit Log لكل إجراء حساس: المستخدم/العميل، الإجراء، الوقت، الفرع، الجهاز (FR-014) |
| لوحة إدارة عربية RTL | React + Vite: دخول، رئيسية، فروع، مستخدمون، أجهزة، هاردوير، طابور طباعة، عملاء API، سجل عمليات — Tajawal، `dir="rtl"` |

### تنظيف الهوية (هذه الجلسة)

- إعادة تسمية كاملة: Restaurant MS → **YAKEBDA MS**، restaurant-ms → `yakebda-ms`، RMS-XX → **YKMS-XX** في الكود والاختبارات والتعليقات وعقد الجسر والحزم والواجهة.
- بادئة الرموز: `rms_` → `ykms_` — قواعد البيانات: `ykms` / `ykms_test` — البريد التجريبي: `owner@ykms.local`.
- الوثائق التاريخية (SRS/Diagrams v1.3) أُبقيت كما هي كمرجع تاريخي فقط.

### التقوية الأمنية (هذه الجلسة)

- عزل حسابات صارم في كل مسارات الجسر: heartbeat، سحب المهام، إبلاغ النتيجة، ack درج الكاش — كلها مقيدة بـ `account_id` الخاص بالـ token.
- heartbeat لا يستطيع تحديث نقاط هاردوير خارج حساب الـ token حتى لو مُررت في القائمة.
- إنشاء نقطة هاردوير يتحقق أن `device_id` يخص **نفس الحساب ونفس الفرع**.
- 7 اختبارات أمنية جديدة (`tests/security.test.ts`) تثبت منع الوصول العابر للحسابات + استمرار عمل الجسر الشرعي.

### الاختبارات

- **21/21 ناجحة** (14 تأسيس + 7 أمنية) على PostgreSQL حقيقي (`ykms_test`).

### القرارات المعمارية

- `docs/adr/ADR-001-tenant-isolation-and-identity.md`
