# الوثيقة الرئيسية للمشروع — YAKEBDA MS v1.0 AR/RTL

**الاسم الرسمي:** YAKEBDA MS
**الإصدار:** v1.0
**اللغة الأساسية:** العربية — **الاتجاه:** RTL
**حالة المشروع:** مشروع مستقل بالكامل عن AKYRO

---

## 1. هوية المشروع

- الاسم الرسمي الوحيد: **YAKEBDA MS**.
- الاسم التقني للمستودع والحزم: `yakebda-ms` و `@ykms/*`.
- تسمية المراحل الرسمية: `YKMS-XX`.
- الاسم القديم **Restaurant MS / restaurant-MS** والتسمية **RMS-XX** أصبحا **تاريخيين فقط** — يظهران حصريًا في وثائق المتطلبات الأصلية (`docs/Restaurant_MS_SRS_v1.3_AR_RTL.md` و `docs/Restaurant_MS_Diagrams_v1.3_AR_RTL.md`) ولا يُستخدمان في أي كود أو وثيقة أو commit جديد.
- بادئة رموز API الرسمية: `ykms_`.

## 2. ما هو YAKEBDA MS

نظام إدارة مطاعم متكامل RMS/POS، عربي أولًا، بواجهة RTL، متوافق مع أجهزة Windows، يشمل: كاشير POS، إدارة الطلبات، الطاولات، شاشة المطبخ KDS، الطباعة، المنيو، الشيفتات، العملاء، المخزون والوصفات، التقارير، و APIs للموقع و QR والتطبيقات، مع طبقة ربط محلية بالهاردوير (Local Device Bridge).

المرجع التفصيلي للمتطلبات: SRS v1.3 (الوثيقة التاريخية) — كل متطلباتها الوظيفية FR-XXX وغير الوظيفية NFR-XXX سارية كما هي، وتُنفذ تحت مراحل YKMS.

## 4. المعمارية الحالية

- **Backend API:** Node.js + TypeScript + Express + Knex + PostgreSQL — كل المسارات تحت `/api/v1`.
- **لوحة الإدارة:** React + Vite — عربية RTL بالكامل، خط Tajawal.
- **عقد الجسر:** `packages/bridge-contract` — أنواع TypeScript مشتركة + `CONTRACT.md` بالعربية.
- **الأمان:** JWT للمستخدمين، PIN للكاشير/الويتر، رموز API بنطاقات scopes تُعرض مرة واحدة وتُخزن hash فقط، RBAC كامل، Audit Log، وعزل صارم بين الحسابات (Tenant Isolation) في كل مسارات الجسر والهاردوير.

## 5. مبادئ حاكمة

1. عربي أولًا و RTL أولًا — أي شاشة جديدة تُصمم وتُختبر بالعربي أولًا.
2. لا يُنفذ أي نطاق خارج المرحلة الجارية.
3. كل إجراء حساس يمر عبر RBAC ويُسجل في Audit Log.
4. فشل الهاردوير (طباعة/درج) لا يوقف أي عملية تشغيلية (NFR-002).
5. أي بيانات تُقرأ أو تُعدّل عبر token يجب أن تكون داخل حساب الـ token نفسه — بلا استثناء.
