# ADR-002 — مصادر الطلب وقوائم الأسعار

**التاريخ:** 2026-07-12  
**الحالة:** Accepted  
**المرحلة:** YKMS-02H

## السياق

نقطة البيع تحتاج تسجيل قناة/مصدر الطلب بصورة إلزامية، مع اختلاف سعر أو إتاحة بعض الأصناف حسب المصدر. تكرار المنتجات والمنيو لكل قناة سيؤدي إلى تضارب الصور والمكونات والتعديلات التشغيلية.

## القرار

- توجد منيو أساسية واحدة للحساب.
- يمثل `order_sources` قنوات البيع التشغيلية.
- يخزن `source_product_rules` فقط إتاحة الصنف والسعر المخصص للمصدر.
- أولوية السعر: سعر المصدر ← سعر الفرع ← السعر الأساسي.
- يحفظ الطلب `source_id` و`source_name_snapshot`.
- يحفظ `order_items.unit_price` و`line_total` كسجل مالي ثابت.
- نقطة البيع تمنع إنشاء الطلب بدون اختيار مصدر.
- خلال فترة الانتقال فقط، طلبات API القديمة التي لا ترسل `source_id` تُنسب إلى المصدر النظامي `direct`.
- الطلبات التاريخية السابقة للهجرة تظل بدون مصدر وتظهر بصيغة «طلب سابق — المصدر غير مسجل».

## الأسباب

- منع تكرار كتالوج المنتجات.
- الحفاظ على مصدر حقيقة واحد للصور والمكونات والخيارات.
- دعم أسعار القنوات مع بقاء الطلبات القديمة ثابتة.
- توفير انتقال آمن لعملاء API الحاليين.

## التأثير

- إضافة هجرتين منطقيتين داخل هجرة واحدة: المصادر، ثم قواعد المنتج للمصدر، ثم مرجع اختياري على الطلب.
- التسعير وإتاحة المنتجات يتحققان على الخادم.
- إعدادات المصادر تستخدم واجهة بطاقات وصفوف responsive بدل جدول عريض.
- يلزم اختبار بصري نهائي لنقطة البيع والإعدادات قبل الدمج.

## البدائل المرفوضة

- نسخ المنتجات والمنيو لكل مصدر.
- حفظ السعر القادم من الواجهة.
- جعل المصدر نصًا حرًا داخل الطلب.
